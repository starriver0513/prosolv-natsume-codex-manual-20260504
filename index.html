Goal

ゴール設計

夏目さんが「何を見ればいいか分からない」「Skillやroutineが古い場所を見ている」「Automationが自分のPCで動かない」を解消する。

1. 自律実行

LINEラリー、事業部タスク、成果物フォルダを見て、SNS/講座/広告の次アクションを自分で切り出し、Codexに実行させる。

2. Skill改善

出力が薄い、古いパスに保存される、文体が機械的などの問題を、Skill本体とセルフチェックに反映する。

3. Automation整備

スタッフPCのCodexにもroutine/Automationを設定し、毎週の確認と日次のSNSリサーチが止まらない状態にする。

Skill

Skill改善のルール

LINEラリーで出ている課題は、X長文・X記事・Threads・Typefully周りのSkillとroutineに反映する。

改善対象

X長文X記事ThreadsTypefullyリサーチ保存

特に x-long-post 相当、X記事、Threads長文、Typefully予約投稿、ナレッジスクレイピングを優先。

直近フィードバック

  • Claude.mdネタの繰り返しを避ける。
  • 「主張の前に実績」の数字パターンを増やす。
  • 丁寧すぎる句点+改行で機械感を出さない。
  • 星川さんの殴り書き感、口癖、過去Xナレッジを反映する。
  • セルフチェックを厚くして、すぐ真似できる要素を入れる。

完了条件

  • Skill本体を変更した。
  • 出力先が現行フォルダになっている。
  • テスト出力を1本以上作った。
  • 旧パス出力がないことを確認した。
  • 変更内容とテスト結果を週次報告にまとめた。
禁止: routineやSkillが古い 旧D番号、01_現活性事業部、アーカイブ配下に出力する状態のまま運用しない。

Prompt

スタッフがCodexに最初に貼るプロンプト

スタッフPCでこのまま貼って、現行状態の診断から始める。

ゴール:
このPCで、星川さんとSyncthing同期している `⭐️AIフォルダ_Prosolv` を正本として開き、SNS運用・AIマーケ講座サポート・広告運用を自律実行できる状態にする。
特に、Skill改善とCodex Automation設定まで終わらせる。

最初に必ず読む:
1. `CLAUDE.md`
2. `directory.md`
3. `タスク管理ルール.md`
4. `📦事業部/SNS事業部/タスク.md`
5. `📦事業部/プロダクト事業部/タスク.md`
6. `📦事業部/リストマーケ事業部/タスク.md`
7. `📦事業部/AI活用事業部/タスク.md`
8. `📦事業部/経営企画/タスク.md`

診断してほしいこと:
- このPCの同期フォルダパスが正しいか
- Skillが現行フォルダを参照しているか
- routine / Codex Automation が旧パスを参照していないか
- X長文、X記事、Threads、Typefully、SNSリサーチの出力先が正しいか
- 週次報告で、成果物URL・変更ファイル・星川確認待ちを出せるか

実行してほしいこと:
1. 旧パス参照を洗い出す
2. 必要なSkill改善案を出す
3. すぐ直せるSkillは直す
4. Automation設定の不足を洗い出す
5. テスト出力を作る
6. 最後に「完了したこと / 未完了 / 星川確認待ち / 次の週次報告文」を出す
Definition of Done

やり切った判定

「理解しました」では終わりません。スタッフPC上で次のチェックが通れば完了です。

  • Syncthing: スタッフPCで ⭐️AIフォルダ_Prosolv が開ける。
  • Codex: 同期フォルダを作業ディレクトリとして開き、正本ファイルを読める。
  • 週次報告: 成果物URL、変更ファイル、テスト結果、星川確認待ちを1つの報告にまとめられる。
  • Skill: X長文・X記事・Threads・Typefully系の少なくとも1つで、改善差分とテスト出力がある。
  • Automation: スタッフPCの ~/.codex/automations/ を確認し、不足分の作成/修正方針が出ている。
  • 旧パスなし: 新しい出力が旧D番号・アーカイブ・過去フォルダに落ちていない。
  • 報告: 完了/未完了/ブロッカー/次アクションが週次報告で共有されている。
